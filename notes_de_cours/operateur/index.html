    Opérateurs logique SQL

    IN

    L'opérateur IN nous permet d'inclure plusieurs valeurs de comparaison dans une clause WHERE. On peut le voir comme plusieurs OR sur une même colonne. On peut aussi l'utiliser avec une sous-requête.

    Syntaxe

    -- Utilisation comme une série de OR
SELECT nom_colonne
FROM ma_table
WHERE nom_colonne_test IN ('Valeur1', 'Valeur2', 'Valeur3', ...);

-- Utilisation avec une sous-requête
SELECT nom_colonne
FROM ma_table
WHERE nom_colonne_test IN (
    sous-requête 
);

    Exemple

    Je veux la liste des athlètes qui sont originaire du Québec, de l'Ontario ou de la Saskatchewan

    SELECT *
FROM athlete a
WHERE a.province IN ('Québec', 'Ontario', 'Saskatchewan');

    À partir des tables suivantes, je veux avoir la liste de tous les athlètes qui ont déjà participer à un marathon. La base de données contient une table athlète qui contient la liste de tous les athlètes et une table résultat qui contient le résultat à une course, le id de l'athlète et le id de l'épreuve (le marathon à un id de 4).

    SELECT *
FROM athlete a 
WHERE a.id IN (
    SELECT r.athlete_id 
    FROM resultat r 
    WHERE r.epreuve_id = 4 
          AND r.athlete_id = a.id
    );

    LIKE

    L'opérateur LIKE nous permet d'utiliser les "jokers" (wildcard) suivant dans une expression pour faire une recherche dans un colonne spécifique avec un certain motif.

    • % remplace n'importe quel texte de n'importe quelle longueur.
    • _ remplace un seul caractère.

    Exemple 

    -- Liste tous les athlètes dont le nom débute par Fré
SELECT * FROM athlete a WHERE a.nom LIKE 'Fré%';

-- Liste tous les athlètes dont le nom se termine par ette
SELECT * FROM athlete a WHERE a.nom LIKE '%ette';

-- Liste tous les athlètes dont le nom contient ch (le terme ch pour être situé au tout  
-- début du nom, dans le milieu ou à la fin)
SELECT * FROM athlete a WHERE a.nom LIKE '%ch%';

-- Liste tous les athlètes dont le nom se débute par Fr, est suivi d'une lettre quelconque --- et se termine par chette.
SELECT * FROM athlete a WHERE a.nom LIKE 'Fr_chette';

    REGEXP

    On peut aussi faire des comparaison beaucoup plus complexe en utilisant l'opérateur REGEXP et des expressions régulières.

    Syntaxe

    SELECT nom_colonne
FROM ma_table
WHERE nom_colonne_test REGEXP (expression_reguliere)

    Exemple

    Je veux la liste des athlètes qui ont un nom d'exactement 9 caractères

    SELECT *
FROM athlete a 
WHERE a.nom REGEXP "^.{12}$";

    Pour créer et tester des expressions régulières : https://regexr.com/

    NOT

    L'opérateur NOT va inverser le résultat d'une condition. Il peut être jumelé à la plupart des autres opérateurs.

    Exemple

    Je veux la liste des athlètes qui ne sont PAS originaire du Québec, de l'Ontario ou de la Saskatchewan

    SELECT *
FROM athlete a
WHERE a.province NOT IN ('Québec', 'Ontario', 'Saskatchewan');

    Je veux la liste des athlètes dont l'adresse courriel n'est pas une adresse GMAIL

    SELECT *
FROM athlete a
WHERE a.courriel NOT LIKE '%@gmail.com';

    Je veux la liste des athlètes dont la valeur de la colonne date_naissance n'est pas NULL

    SELECT *
FROM athlete a
WHERE a.date_naissance IS NOT NULL

    EXISTS

    L'opérateur EXISTS est utilisée pour déterminer si le résultat d'une sous-requête retourne des enregistrements ou non. Il retournera TRUE si la sous-requête contient un ou plusieurs enregistrement et FALSE dans le cas contraire. Le traitement se termine dès qu'une ligne correspondante dans la sous-requête est trouvée.

    Syntaxe

    SELECT nom_colonne
FROM ma_table
WHERE EXISTS(sous-requête);

    Exemple

    À partir des tables suivantes, je veux avoir la liste de tous les athlètes qui ont déjà participer à un marathon. La base de données contient une table athlète qui contient la liste de tous les athlètes et une table résultat qui contient le résultat à une course, le id de l'athlète et le id de l'épreuve (le marathon à un id de 4).

    select *
from athlete a 
where exists (
    select 1 
    from resultat r 
    where r.epreuve_id = 4 
          and r.athlete_id = a.id
    );

    Il est aussi intéressant de noter ici que même si un athlète a participé à plus d'un marathon il ne sera affiché qu'une fois dans la liste car comme dit plus haut, dès qu'il y a une correspondance, la sous-requête de la fonction se termine.

    L'opérateur EXISTS peut aussi être utilisée dans la clause WHERE d'un UPDATE, INSERT et DELETE.

    DELETE FROM athlete
WHERE EXISTS(
    select 1 
    from resultat r 
    where r.epreuve_id = 4 
          and r.athlete_id = a.id
    );

    Dans cette exemple, tous les athlètes qui ont déjà participé à un marathon seront supprimées.

    On peut aussi utilisé l'opérateur NOT pour inverser le résultat. Si on reprends l'exemple où on liste les athlètes qui ont déjà participer à un marathon mais qu'on voudrait connaitre l'inverse, c'est-à-dire d'avoir la liste des athlètes qui n'ont jamais participé à un marathon. Il suffit d'ajouter NOT devant le EXISTS pour inverser le résultat et avoir la réponse recherché.

    select *
from athlete a 
where not exists (
    select 1 
    from resultat r 
    where r.epreuve_id = 4 
          and r.athlete_id = a.id
    );
