    Épreuve finale - partie 1 - Formatif

    Consignes

    • Vous avez droit à toutes vos notes et Internet.
    • Aucune communication n'est permise (messagerie, courriel, etc.). Un élève pris en flagrant délit se verra attribuer la note de 0 pour plagiat.
    • La durée de l'examen est de 1h50.
    • Une fois l’examen terminé, vous devez me remettre les fichiers suivants dans un dossier compressé nommé au format suivant : NomPrenom_ExamenFinalPartie1.
    • Un diagramme de classe en format image.
    • Un script SQL de création de la base de données et des tables.
    • Un script SQL de création des déclencheurs.
    • Un script SQL de création d'un usager.
    • Le dossier compressé doit être remis sur Teams dans le devoir de l'examen.
    • Il est de votre devoir de vous assurer que tous les fichiers sont inclus dans la remise. Aucun fichier ne sera accepté une fois que vous aurez quitté le lieu de l'examen.

    Mise en situation

    L'entreprise qui vous emploie développe une application qui permet de créer des questionnaires et vous êtes en charge de créer la base de données qui contiendra ces questionnaires. Vous avez reçu de votre chargé de projet les écrans suivants pour vous aider dans votre travail.

    Ce premier écran représente une vue du questionnaire dans son ensemble. Pour chaque questionnaire on doit obligatoirement définir un titre et on a l'option d'ajouter un court texte descriptif. On peut ensuite ajouter une ou plusieurs questions qui peuvent être de deux types : à choix de réponse ou bien au format texte. Chaque type à ses propres particularités.

    Le questionnaire à une colonne qu'on ne voit pas à l'écran et qui contient le pointage cumulé de toutes les questions.

    Questions à choix de réponse.

    Voici une capture d'écran des options d'une question à choix de réponse.

    • La question peut avoir plusieurs choix, mais seulement un de ces choix est la bonne réponse.
    • On peut ajouter un pointage ou pas à la question.
    • La question peut être obligatoire ou non.

    Questions au format texte

    Voici une capture d'écran d'une question au format texte.

    - Pour ce type de question, l'usager doit saisir sa réponse en texte. On peut avoir une ou plusieurs bonnes réponses. - On peut ajouter un pointage ou pas à la question. - La question peut être obligatoire ou non.

    Directives

    Question #1

    Vous devez créer un modèle de la base de données qui permettra de reproduire des questionnaires selon les spécifications de la mise en situation plus haut. Le modèle doit seulement permettre de sauvegarder les informations de création du questionnaire, vous ne devez pas à gérer les réponses des participants.

    Votre modèle doit être sous la forme d'un diagramme de classes et répondre aux points suivants :

    • Votre diagramme doit répondre à la convention de nomenclature telle que décrite dans les notes de cours "Analyse et modélisation de systèmes".
    • Pour chaque colonne de chaque table du diagramme, vous devez spécifier un type de données adéquat, si elle est une clé primaire (PK) ou étrangère (FK). Si la colonne est ni une clé primaire ou étrangère, mais qu'elle ne peut être nulle, vous devez aussi l'indiquer (NOT NULL).
    • ex. #1 -> id INT PK
    • ex. #2 -> code_usager VARCHAR(50) NOT NULL
    • Pour les relations entre les tables, vous devez m'indiquer le type de relation en utilisant la notation 1<--->1 pour une relation d'un à un et 1<--->n ou n<--->1 pour une relation d'un à plusieurs
    • Votre modélisation doit répondre aux règles de la troisième forme normale.

    Vous devez me fournir votre diagramme de classes dans un format image (jpg, png) et le nommer NomPrenom_diagramme

    Question #2

    Créez un script SQL qui va créer la base de données ainsi que toutes les tables représentées dans votre diagramme de classes.

    • La base de données doit se nommer ExamenFinal_NomPrenom
    • Dans la création des tables, vous devez inclure la création des clés primaires et étrangères s'il y a lieu.
    • Le script doit pouvoir être lancé plus d'une fois sans provoquer d'erreurs.
    • Chaque lancement du script doit régénérer la base de données ainsi que les tables.

    Le script SQL à remettre doit se nommer NomPrenom_ceationBD.sql.

    Question #3

    Créez un script SQL qui va ajouter des déclencheurs (Triggers) qui permettront de recalculer la colonne du pointage cumulé du questionnaire à chaque fois que vous allez ajouter, modifier ou supprimer une question.

    • Pour chacune des opérations (ajout, modification et suppression), choisissez le bon moment pour placer le déclencheur.
    • Si la question n'a pas de pointage, vous ne devez pas faire la mise à jour de la colonne.
    • Le script doit pouvoir être lancé plus d'une fois sans provoquer d'erreurs.
    • Chaque lancement du script doit recréer les triggers.

    Le script SQL à remettre doit se nommer NomPrenom_creationTrigger.sql.

    Question #4

    Créez un script SQL qui fera la création d'un usager et lui donnera des permissions selon les informations suivantes:

    a) Informations de l'usager :

    • nom : correcteur
    • mot de passe : !3ct3Ur
    • hôte accepté : localhost

    b) Permission de l'usager :

    • Il peut faire des requêtes SELECT et UPDATE sur toutes les colonnes de la table qui contient les informations générales du questionnaire de votre base de données sauf la colonne du pointage calculé.
    • Pour toutes les autres tables de la base de données, il peut faire des requêtes SELECT et UPDATE.

    Le script doit pouvoir être lancé plus d'une fois sans provoquer d'erreurs et chaque lancement du script doit recréer les triggers.

    Le script SQL à remettre doit se nommer NomPrenom_creationUsager.sql.
